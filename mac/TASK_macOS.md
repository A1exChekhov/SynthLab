## 0. Контекст и эталон

В этом же репозитории лежит **рабочая Windows-версия**: `channel-splitter/splitter_gui.py`.
Используй её как эталон функций и UX. Что она умеет (нужно повторить на macOS):

- Захват системного звука (на Win — через VB-CABLE / WASAPI loopback).
- **Динамический список выходных колонок** (добавить/удалить). У каждой:
  устройство, **роль/канал** (Левый / Правый / Моно), громкость, **SUB** (сабвуфер с
  низкочастотным кроссовером), тест колонки, индикатор уровня.
- Несколько **источников**, у каждого: громкость, баланс (панорама между колонками),
  инверсия фазы, mute.
- **12-полосный эквалайзер** (20 Гц…20 кГц) со спектром, пресеты.
- Эффекты: **BASS / SPATIAL / 3D / 7.1 SURROUND** (каждый с вкл/выкл).
- Транспорт ON/OFF, мастер-громкость.
- Бренд **Errarium™** (шапка/футер/метаданные), лицензия.

macOS-версия должна это повторить и добавить: **многоканальный декод (5.1/7.1),
матрицу ролей под домашний кинотеатр, авто-выравнивание задержек (калибровка
микрофоном), drift-коррекцию**.

## 1. Архитектура (важные решения)

### 1.1 Вывод на несколько устройств
`AVAudioEngine` подключается к **одному** выходному устройству. Для нескольких колонок —
два варианта, выбрать по ходу:

**Вариант A (рекомендуется): Aggregate Device.**
Программно создать через Core Audio HAL **агрегатное устройство**, объединяющее выбранные
выходы в одно много-канальное. Плюсы:
- единые часы для всех суб-устройств;
- встроенная **Drift Correction** Core Audio (решает «уплывание» BT-часов почти даром —
  включать флаг `kAudioSubDeviceDriftCompensation` на суб-устройствах, кроме мастер-часов).
Маршрутизация программных каналов → физические каналы агрегата делается матрицей/каналмэпом.

Ключевые Core Audio вызовы: `AudioHardwareCreateAggregateDevice` (через
`kAudioPlugInCreateAggregateDevice`), словарь описания с
`kAudioAggregateDeviceSubDeviceListKey`, `kAudioAggregateDeviceMasterSubDeviceKey`,
`kAudioSubDeviceDriftCompensationKey`. Удалять — `AudioHardwareDestroyAggregateDevice`.

**Вариант B (запасной): по одному `AVAudioEngine` на колонку.**
Каждый движок: `AVAudioSourceNode` (рендер-колбэк, тянет из общего ring-buffer) → `AVAudioUnitEQ`
→ `AVAudioUnitDelay` → `mainMixerNode` → `outputNode`, у которого через AudioUnit-свойство
`kAudioOutputUnitProperty_CurrentDevice` выставлено нужное устройство. Проще в реализации,
ближе к Windows-версии, но часы у устройств свои → синхронизацию держать калибровкой/делеями.

> Начать можно с варианта B (быстрее завести), затем при необходимости перейти на A ради синхро.

### 1.2 Захват источника (системный звук)
Аналог VB-CABLE на macOS:
- **BlackHole** (open-source, лицензия **MIT — можно легально бандлить**) — виртуальное
  аудиоустройство. Юзер шлёт системный звук в BlackHole, приложение читает его как вход
  (`AVAudioEngine.inputNode` с выбранным устройством, или AUHAL input).
- **ScreenCaptureKit Audio** (macOS 13+) — захват системного звука **без драйвера**
  (`SCStream` с `capturesAudio = true`). Для v1 достаточно BlackHole; ScreenCaptureKit — как
  второй режим «без установки драйвера».
- Плюс возможность выбрать любой обычный вход/файл как источник.

### 1.3 Декод многоканальности (5.1/7.1)
Для домашнего кинотеатра — принять много-канальный поток (Dolby Digital/DTS/AAC 5.1/7.1):
- системный `AudioToolbox` (`AudioFile`/`ExtAudioFile`/`AVAudioFile`) для файлов;
- либо интеграция **FFmpeg (libavcodec)** для потоков/контейнеров.
Декодер раскладывает в массив независимых буферов: L, R, C, LFE, Ls, Rs (+ при 7.1 Lb, Rb).

### 1.4 Матрица ролей (сердце приложения)
Сопоставление программных каналов (L/R/C/LFE/Ls/Rs) с физическими каналами конкретных
колонок. Реализация: матричная маршрутизация — `kAudioUnitSubtype_MatrixMixer` (AUMatrixMixer)
или ручной микс в рендер-колбэке `AVAudioSourceNode`. Пользователь задаёт роль каждой колонки;
программа строит матрицу. Примеры:
- LFE (бас) → лев+прав каналы сабвуфера;
- L/R → фронтальная пара;
- Ls/Rs → тыловые колонки.

### 1.5 Эквалайзер и эффекты
- **EQ:** `AVAudioUnitEQ` (отдельный экземпляр на направление/колонку). 12 полос — `bands`
  с типом `.parametric`/`.lowShelf`/`.highShelf`. Для саба — low-pass на EQ. Спектр —
  через tap + Accelerate FFT.
- **Эффекты** (как в Win): BASS (low-shelf), SPATIAL (M/S ширина), 3D (Haas/кроссфид через
  `AVAudioUnitDelay`/микс), 7.1 SURROUND (ранние отражения — короткие задержки).

### 1.6 Задержки и синхронизация (ГЛАВНЫЙ ВЫЗОВ)
У разных BT-колонок разная внутренняя задержка (JBL быстрее, Sony RA5000 с 360 Spatial —
большой внутренний DSP-буфер). Нужна **линия задержки на каждый выход**:
- `AVAudioUnitDelay` (feedback=0, wetDryMix=100) или собственный кольцевой буфер;
- в UI рядом с каждой колонкой — ползунок **«Задержка, мс»**;
- общая задержка системы сообщается наружу (чтобы видеоплеер сдвигал картинку).

## 2. Автокалибровка задержек (обязательно в этой версии)

Алгоритм (выполняется кодом приложения):

1. **Свип-тон (Chirp):** поочерёдно в каждую колонку — логарифмический свип 20 Гц→20 кГц
   за 1–2 с. Уникален, не путается с шумом комнаты.
2. **Запись отклика:** одновременно открыть запись со встроенного микрофона Мака
   (`AVAudioEngine.inputNode` / AUHAL input), писать в буфер.
3. **Кросс-корреляция:** сопоставить эталонный свип `x(t)` и запись `y(t)`:
   `R_xy(τ) = Σ x(t)·y(t+τ)`. Пик функции = абсолютная задержка прохождения сигнала (в сэмплах).
   Считать через **Accelerate.framework / vDSP** (`vDSP_conv`, FFT) — быстро на Apple Silicon.
4. **Дельты и Delay Lines:** найти `τ_max` среди колонок; для каждой выставить
   `delay = τ_max − τ_i`. Пример: JBL τ=120 мс, Sony τ=280 мс → JBL получает задержку 160 мс,
   Sony — 0. Звук из обеих колонок доходит до ушей одновременно.

UI: кнопка **«Калибровка»** (прогон свипов + расчёт), показ измеренных задержек,
кнопка **«Быстрая подстройка»** для повторного замера (борьба с дрейфом).

### Дрейф часов (Clock Drift)
У каждой BT-колонки свой кварц → за 1–1.5 ч задержка «уплывает» на 10–30 мс.
Решения:
- Aggregate Device + `kAudioSubDeviceDriftCompensation` (Core Audio сам поджимает сэмплы);
- либо периодическая «быстрая подстройка» калибровкой;
- либо собственная ресемпл-коррекция по системному таймеру.

## 4. Настройка проекта в Xcode
1. New Project → **macOS → App** → SwiftUI, Swift. Название: `ChannelSplitter`, организация: `Errarium`.
2. **Info.plist:**
   - `NSMicrophoneUsageDescription` = "Для автокалибровки задержек колонок приложение
     измеряет звук через микрофон." (без этого микрофон не даст доступ — калибровка не заработает).
3. **Signing & Capabilities:**
   - App Sandbox: включить **Audio Input** (микрофон). Если читаешь чужие аудиоустройства/агрегат —
     возможно отключить sandbox или добавить нужные исключения; для распространения вне App Store
     проще без sandbox + Developer ID + нотаризация.
4. **Frameworks (Link Binary):** AVFoundation, CoreAudio, AudioToolbox, Accelerate,
   ScreenCaptureKit (опц.), CoreMedia.
5. Подключить файлы из `mac/ChannelSplitterMac/Sources/` (их добавит следующая сессия кодом).

## 6. Подводные камни (учесть в коде)
- **Пропускная способность Bluetooth:** классический BT-чип Мака захлёбывается на 4–5
  одновременных несжатых потоках на разные устройства → заикания. Рекомендация в UI:
  не более 2–3 BT-устройств одновременно; тяжёлые колонки (Sony RA5000) подключать по **Wi-Fi
  (AirPlay/DLNA)**, а задержки выравнивать калибровкой — это разгружает 2.4 ГГц.
- **AirPlay/Wi-Fi вывод** программно ограничен API — заложить как отдельный режим/фазу.
- **Разрешение микрофона** — без `NSMicrophoneUsageDescription` калибровка молча не работает.
- **Sandbox** может блокировать доступ к произвольным аудиоустройствам — тестировать раздачу
  прав; для дистрибуции — Developer ID + нотаризация (а не App Store, из-за низкоуровневого аудио).

## 7. Фазы реализации
1. **Ядро вывода:** список колонок → вывод (вариант B: движок на колонку) + роль/громкость/mute,
   захват источника (BlackHole), ON/OFF, мастер. Проверить звук на 2 колонках.
2. **EQ + эффекты + сабвуфер (кроссовер)** + спектр + пресеты.
3. **Задержки:** ручные ползунки + **автокалибровка** (chirp + микрофон + vDSP) + «быстрая подстройка».
4. **Многоканальный декод 5.1/7.1** + матрица ролей (домашний кинотеатр).
5. **Aggregate Device + Drift Correction** (синхро) — переход с варианта B на A при необходимости.
6. **Wi-Fi/AirPlay** вывод (опц.), авто-акустическая коррекция (расширение калибровки на АЧХ).
7. **Сборка:** .app, Developer ID signing + нотаризация, .dmg. Бренд/иконка Errarium™.

## 8. Критерии готовности v1
- Системный звук раздаётся на ≥2 выбранных устройства с разными ролями.
- На каждую колонку работают: громкость, EQ, ручная задержка, тест-сигнал, индикатор.
- Автокалибровка измеряет задержки микрофоном и выставляет delay lines (щелчок/свип во все
  колонки слышен одновременно).
- Нет заиканий на 2 BT-колонках; есть предупреждение при добавлении 4+.
- Тёмный UI с брендингом Errarium™, паритет фич с Windows-версией (кроме многоканала — это фаза 4).
